## Intro

A **unit test** is a small piece of code that tests a **specific function** or **module** in your app.

You should write unit tests for the functions that have complex logic, lots of edge cases, critical to the app, etc.

## Basics

**Test Suite**: Group of related tests for a single function/module
**Test Case**: Tests a single behavior of the function 
**Assertion**: Asserts a certain outcome or result

```JS
export function longestString(a: string, b: string) {
  return a.length >= b.length ? a : b;
}
```

```JS
describe('longestString', () => {
  it('returns the longer string when second is longer', () => {
    const result = longestString('a', 'ab');
    expect(result).toBe('ab');
  });

  it('returns the longer string when first is longer', () => {
    const result = longestString('hello', 'hi');
    expect(result).toBe('hello');
  });

  it('returns first string when both have same length', () => {
    const result = longestString('cat', 'dog');
    expect(result).toBe('cat');
  });

  it('works with empty strings', () => {
    const result = longestString('', 'a');
    expect(result).toBe('a');
  });
  
  // Note that this case will fail, because we didn't trim
  it('handles empty spaces', () => {
	  const result = longestString('     a', 'bc');
	  expect(result).toBe('bc');
  })
});
```
