
## Epoch Time

- **Epoch time** (Unix time) is the number of seconds since January 1, 1970, 00:00:00 UTC.

## Date Constructor

- The `Date` object is a built-in JavaScript object for creating and managing dates and times.
- Returns a `Date` object containing date, time, and timezone information.
- **Static Method**: `Date.now()`
    - Returns the number of milliseconds elapsed since January 1, 1970, 00:00:00 UTC.
    - Useful for benchmarking or timing operations.

### Example

```javascript
console.log(Date.now()); // e.g., 1706994310194
```

## Get Date and Time

- Create a `Date` object to access current or specific date/time information.
- Methods to retrieve specific components:
    - `getDate()`: Day of the month (1–31).
    - `getTime()`: Milliseconds since January 1, 1970, 00:00:00 UTC.
    - `getDay()`: Day of the week (0–6, 0 = Sunday).
    - `getMonth()`: Month (0–11, 0 = January).
    - `getFullYear()`: Four-digit year.
    - `getHours()`: Hour (0–23).
    - `getMinutes()`: Minutes (0–59).
    - `getSeconds()`: Seconds (0–59).
    - `getMilliseconds()`: Milliseconds (0–999).

### Example

```javascript
let date = new Date();
let birthday = new Date("Jun 11 2003");
let dateDif = date - birthday;
console.log(dateDif / 1000 / 60 / 60 / 24 / 365); 
// Approximate years

console.log(date.getDate()); // e.g., 16
console.log(date.getTime()); // e.g., 1706994310194
console.log(date.getDay()); // e.g., 3 (Wednesday)
console.log(date.getMonth()); // e.g., 6 (July)
console.log(date.getFullYear()); // e.g., 2025
console.log(date.getHours()); // e.g., 14
console.log(date.getMinutes()); // e.g., 51
console.log(date.getSeconds()); // e.g., 0
console.log(date.getMilliseconds()); // e.g., 123
```

## Set Date and Time

- Methods to modify a `Date` object:
    - `setTime(milliseconds)`: Sets the date/time as milliseconds since January 1, 1970.
    - `setDate(day)`: Sets the day of the month (1–31). Negative/positive values shift to previous/next months.
    - `setFullYear(year, month, day)`: Sets year, optional month (0–11), and day (1–31).
    - `setMonth(month, day)`: Sets month (0–11), optional day (1–31).
    - `setHours(hours, minutes, seconds, ms)`: Sets hours (0–23), optional minutes (0–59), seconds (0–59), milliseconds (0–999).
    - `setMinutes(minutes, seconds, ms)`: Sets minutes (0–59), optional seconds (0–59), milliseconds (0–999).
    - `setSeconds(seconds, ms)`: Sets seconds (0–59), optional milliseconds (0–999).

- Negative values move to the previous unit (e.g., day, month); values exceeding the range move to the next unit (e.g., `setHours(25)` sets 1:00 AM the next day).

### Example

```javascript
let dateNow = new Date();
dateNow.setTime(0); 
// Sets to January 1, 1970, 00:00:00 UTC (plus timezone offset)
```

## Date.parse

- `Date.parse(dateString)`: Returns the number of milliseconds between the specified date and January 1, 1970, 00:00:00 UTC.

### Example

```javascript
console.log(Date.parse("11 Jun 2003")); // 1055278800000
```

## Creating Date Objects

- `new Date(timestamp)`: Creates a `Date` object from milliseconds since epoch.
- `new Date(dateString)`: Creates a `Date` object from a date string.
- `new Date(year, month, day, hours, minutes, seconds)`: Creates a `Date` object from numeric values.

### Supported Date String Formats

- String formats:
    - `"Oct 25 1982"`
    - `"10/25/1982"`
    - `"1982-10-25"` (ISO standard)
    - `"1982 10"`
    - `"1982"`
    - `"82"`
    - `"1982-10-25T06:10:00Z"` (includes timezone; without `Z`, uses local timezone)
- Numeric format (no quotes):
    - `1982, 9, 25, 2, 10, 0` (year, month [0–11], day, hours, minutes, seconds)
    - `1982, 9, 25` (year, month, day)

### Example

```javascript
let myDate = new Date(1055278800000); // From timestamp
console.log(myDate); // Wed Jun 11 2003 00:00:00 GMT+0000

let dateStr = new Date("1982-10-25");
console.log(dateStr); // Mon Oct 25 1982 00:00:00 GMT+0000

let dateNum = new Date(1982, 9, 25, 2, 10, 0);
console.log(dateNum); // Mon Oct 25 1982 02:10:00 GMT+0000
```

## Tracking Operations Time

- Measure execution time by capturing timestamps before and after an operation and calculating the difference in milliseconds.
- Using `Date.now()`:

```javascript
let startTime = Date.now();
for (let i = 0; i < 100000; i++) { /* some operation */ }
let endTime = Date.now();
console.log(`Operation took ${endTime - startTime} ms`);
```

- `Date.now()` is simple but less precise (millisecond resolution) and tied to the system clock, which may be adjusted, leading to potential inaccuracies.[](https://medium.com/%40AlexanderObregon/getting-accurate-time-with-javascript-performance-now-ccd658a97ab3)

## Performance.now()

- Part of the **Performance API**, `performance.now()` provides a high-resolution timestamp (microsecond precision, up to 5µs) relative to the page load or Node.js process start.
- Monotonically increasing, unaffected by system clock changes, making it ideal for benchmarking.
- Syntax: `performance.now()`
- Returns: A `DOMHighResTimeStamp` (milliseconds with microsecond precision, e.g., `563.247`).

### Example

```javascript
const t0 = performance.now();
for (let i = 0; i < 100000; i++) { /* some operation */ }
const t1 = performance.now();
console.log(`Operation took ${t1 - t0} milliseconds.Scroll`)
```
