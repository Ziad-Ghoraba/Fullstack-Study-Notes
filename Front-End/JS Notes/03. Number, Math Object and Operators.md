
## Numbers

- Double precision (64-bit, 16 decimal places)
- Syntactic sugar `_`: `console.log(1_000_000)` → 1000000
- `Number.MAX_SAFE_INTEGER`, `Number.MAX_VALUE`

- Methods: `toString()`, `toFixed(n)` Note that `toFixed` returns a string , `parseInt()`, `parseFloat()`, `isInteger()`, `isNaN()`

```js
console.log((100).toString()); // "100"
console.log(parseInt("100 osama")); // 100
console.log(Number.isInteger("100")); // false
```

### Math Object

- `round()`, `ceil()`, `floor()`, `min()`, `max()`, `pow()`, `random()`, `trunc()`

```js
console.log(Math.round(99.4)); // 99
console.log(Math.min(10, 345, -200, -3456)); // -3456
let randomInt = Math.floor(Math.random() * 10) + 1; // Random 1-10
```

## Operators

- `==` (value only), `===` (value and type)
- `!=`, `!==`
- Logical: `&&`, `||`, `!`
- Unary: `+`, `-` (converts string to number) .. the negative return a number if there's a number and negates it ... also you can use the Number constructor to convert to number.

- Type coercion: Implicit/explicit conversion

```js
console.log(+"100"); // 100
console.log(Number("100")); // 100
console.log(10 == "10"); // true
console.log(10 === "10"); // false
```

   
**(||) — Handles All Falsy Values**

- Used to provide a default when the value is falsy (`null`, `undefined`, `0`, `false`, `NaN`, `""`)

**`??` (Nullish Coalescing) — Only for `null` or `undefined`**

```JS
console.log(`The price is ${price || 200}`);
console.log(`The price is ${price ?? 200}`);
```
