
## Set Data Type

A `Set` is an object that stores unique values, meaning it does not allow duplicate elements. It cannot be accessed by index, and keys are the same as values (keys is an alias for values).

### Properties

- **size**: Returns the number of values in the Set.

### Methods

- **add**: Adds a new element to the Set.
- **delete**: Removes an element from the Set. Returns `true` if the item was found and deleted, `false` otherwise.
- **clear**: Removes all elements from the Set.
- **has**: Returns a boolean indicating whether an element exists in the Set.
- **keys**: Returns a new iterator object containing the values for each element (alias for `values`).
- **values**: Returns a new iterator object containing the values for each element.
- **entries**: Returns a new iterator object containing an array of `[value, value]` for each element.

### Iterator in JavaScript

An iterator is an object that provides a `next()` method, which returns the next item in a sequence. The `next()` method returns an object with two properties:

- **value**: The next item in the sequence.
- **done**: `true` if the last item has been consumed, `false` otherwise.

Example:

```javascript
let mySet = new Set(["a", "b", "c"]);
let iterator = mySet.values();

console.log(iterator.next()); // { value: 'a', done: false }
console.log(iterator.next()); // { value: 'b', done: false }
console.log(iterator.next()); // { value: 'c', done: false }
console.log(iterator.next()); // { value: undefined, done: true }
```

### Printing the Last Item in a Set

To print the last item in a Set:

```javascript
console.log(Array.from(setOfNumbers).pop());
```

## Weak Set Data Type

A `WeakSet` is a special kind of Set that only contains objects as its elements. It allows elements to be garbage collected when there are no other references to them, hence the term "Weak."

### Characteristics

- Not iterable (cannot use methods like `forEach`).
- Only has the following methods: `add`, `has`, `delete`.
- Does not have `clear`, `size`, `keys`, `values`, or `entries`.

## Map Data Type

A `Map` is a collection of key-value pairs where any value (functions, objects, primitives) can be used as a key or value.

### Differences Between Map and Object

- **Key Types**:
    - **Object**: Keys can only be strings or symbols.
    - **Map**: Keys can be any value (functions, objects, primitives).

- **Order of Elements**:
    - **Object**: No guaranteed order.
    - **Map**: Elements are ordered by insertion order.

- **Size**:
    - **Object**: No built-in `size` property.
    - **Map**: Has a `size` property.

- **Iteration**:
    - **Object**: Not directly iterable; requires `Object.keys()` or `Object.values()` to get arrays.
    - **Map**: Iterable with methods like `forEach`, `keys`, `values`, and `entries`.

- **Performance**:
    - **Map**: Better for frequent additions/removals of key-value pairs due to its hash table implementation.
    - **Object**: Less efficient for frequent changes.

### Default Keys

- **Object**: Inherits default keys from `Object.prototype` (e.g., `toString`, `hasOwnProperty`, `valueOf`).
    - To create an object without default keys: `let myObj = Object.create(null);`.
    - In objects, all keys are converted to strings, so `{"10": string, 10: number}` results in a single key-value pair.

- **Map**: Does not have default keys.

### Properties

- **size**: Returns the number of key-value pairs in the Map.

### Methods

- **set**: Adds or updates a key-value pair.
- **get**: Returns the value associated with a key.
- **has**: Returns a boolean indicating whether a key exists.
- **delete**: Removes a key-value pair. Returns `true` if the item was found and deleted, `false` otherwise.
- **clear**: Removes all key-value pairs.
- **keys**: Returns a new iterator object containing the keys.
- **values**: Returns a new iterator object containing the values.
- **entries**: Returns a new iterator object containing an array of `[key, value]` for each element.

### Creating a Map from an Object

To create a Map from an object:

```javascript
let myMap = new Map(Object.entries(myInfo));
```

## WeakMap Data Type

A `WeakMap` is similar to a Map but only allows objects as keys, and these keys do not prevent garbage collection when there are no other references to them.

### Characteristics

- **Keys**: Must be objects.
- **Methods**: Only supports `set`, `get`, `has`, `delete`.
- **Not Iterable**: Does not have `size`, `clear`, `keys`, `values`, or `entries`.
