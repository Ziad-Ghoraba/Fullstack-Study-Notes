A **delegate chain** (or multicast delegate) is a **single delegate variable** that holds **multiple methods**, not just one.

When you call the delegate, **all methods in the chain execute in order**.

```CS
public delegate void LogHandler(string message);

void LogToConsole(string msg) => Console.WriteLine("Con: " + msg);
void LogToFile(string msg) => Console.WriteLine("File: " + msg);

LogHandler logger = LogToConsole;
logger += LogToFile;

logger("System started");

// Output
Con: System started
File: System started
```
***
 **Key rules**:
 
1. **Only delegates with `void` return type should be multicast.**  
    If they return something, only the **last method’s return value** is kept.  
    (The others’ returns are ignored.)
2. If one method in the chain throws an exception, the chain stops unless you handle it.

```CS
delegate int Del(int input);

internal class Program{
	static void Main(string[] args) {
		Del myMathDelegate = Add;
		myMathDelegate += Mul;

		int number = myMathDelegate(5);
		Console.WriteLine(number); 
		// Outputs: 50
		// Only the last method’s return value is kept
	}

	public static int Add(int num) => num + 10;
	public static int Mul(int num) => num * 10;
}
```
***
Generic delegates are **templates** for delegates, just like generic classes.

They let you create a delegate without defining a new delegate type every time.

```CS
delegate T Del<T>(T x, T y);

Del<int> myDelegate = Sum;
```
